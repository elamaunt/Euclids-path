# 29. ProductHall / SteeringEngine — локальный pump без циркулярного payment

> Источник: step00_producthall_steering_pump_strict_ru_2026-07-01.md. Lean: `Engine/ProductHall.lean`
> (`productHall_engine` доказан, без аксиом). Аудит: tools/RESULTS_producthall_audit.md.

## Главное улучшение

Прошлые pump-файлы упирались в **циркулярность** `same_key_payment` («одинаковый ключ ⟹ платёж по
определению»). Этот файл её **явно отвергает** (§9) и заменяет на **4-случайную дихотомию зон**
Legal/Forbidden с настоящим логическим содержанием.

## Доказанное ядро (Lean, без аксиом)

`productHall_engine` (Lemma 8.1): локальный fan-in `X₁→Y`, `X₂→Y`, `X₁≠X₂`, одинаковый паспорт,
`Legal Y`. Разбор зон:
- **оба legal** ⟹ `uniqueLegalLift` ⟹ `X₁=X₂` ⟹ противоречие;
- **смешанные** ⟹ steering (пересечение boundary);
- **оба forbidden** + `Legal Y` ⟹ forbidden-inflow ⟹ steering.

Во всех случаях — `EuclideanEngine`. Это **чистая логика по случаям** (не циркулярна).

## Вердикт аудита (3 агента, conf 0.88): улучшение гигиены, НЕ закрытие

**Циркулярность `same_key_payment` устранена** ✅ (4-случайная логика честна, компилируется). Но
**дыра переехала в `UniqueLegalLift`, и она в ловушке — ТРИЛЕММА:**

ProductHall (§7) требует `a₁≠a₂`, `a₁≡a₂ (mod P_A)`, И `pass X₁ = pass X₂`. Паспорт обязан
одновременно:
- **различать** `a₁,a₂` (иначе UL ложна);
- **НЕ различать** (иначе ProductHall пуст).

Выход один — объявить паспорт «нормальной формой, дающей единственность» = **вшить вывод** (циркулярно).
Аудит: *coarse ⟹ UL ложно / fine ⟹ узел пуст / tautological ⟹ circular*. Решающий факт: grep по
репо — `Passport`/`UniqueLegalLift` есть ТОЛЬКО в этом файле, БЕЗ конструкции. UL — пустая дыра.

**`steering_is_euclidean`** — self-destruct как у standing-pump: `Engine := …∨Steering` по
определению ⟹ тривиально, но высотный EPMI fan-in не опровергает. Нужен новый EPMI — не доказан.

**Несовместимость с fan-in 570→1:** конечная нормальная форма, инъективная для UL, не реализует
many-to-one 570→1. Прямое противоречие с мотивацией узла.

## Что нужно для замыкания (§13)

1. **`UniqueLegalLift`** — должна следовать из паспорта как **канонической нормальной формы** legal-
   lift над lower base. ВАЖНО (файл признаёт): если паспорт = только `a mod P_A`, UL **ложно**; нужна
   настоящая нормальная форма, не вшивающая вывод.
2. **`SteeringEngine ⟹ EuclideanEngine`** — новый EPMI; проверить, не самоликвидируется ли (как
   standing-pump: если `Engine := … ∨ Steering` по определению, то доказанный высотный EPMI его не
   опровергает).

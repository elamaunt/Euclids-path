# Числовая проверка four-corner (distribution-free, точная целочисленная арифметика)

Скрипты: `fourcorner_from_csv.py` (из исходных CSV), `fourcorner_harness.py` (независимое сито).

## Главный результат (подтверждён независимым ситом)

Four-corner **`N₀₀·N₃₃ < N₀₃·N₃₀` выполняется на КАЖДОМ блоке** k=21…28 (и для всех m, и для old-free).
Это отрицательная ассоциация рангов r₋,r₊ (под независимостью было бы равенство).

| k | R_fc=N₀₀N₃₃/N₀₃N₃₀ (old-free) | R_sc=N₀₃N₃₀/N₀₀² | N₃₃/N₀₀ |
|---|---|---|---|
| 21 | 0.876 | 0.0145 | 0.0125 |
| 22 | 0.918 | 0.0121 | 0.0111 |
| 23 | 0.920 | 0.0151 | 0.0139 |
| 24 | 0.976 | 0.0131 | 0.0128 |
| 25 | 0.951 | 0.0145 | 0.0137 |
| 26 | 0.962 | 0.0149 | 0.0144 |
| 27 | 0.972 | 0.0141 | 0.0137 |
| 28 | 0.977 | 0.0162 | 0.0158 |

## Ключевой вывод
`N₃₃/N₀₀ = R_fc · R_sc`. `R_fc → 1` (запас four-corner тает), но `R_sc ≈ 0.014` устойчиво мало.
Значит для `N₃₃ < N₀₀` достаточно НАПРАВЛЕНИЯ four-corner (`R_fc ≤ 1`) + лёгкого `R_sc < 1`.
Трудность сведена к одному направленному корреляционному неравенству `N₀₀N₃₃ ≤ N₀₃N₃₀` — БЕЗ распределения простых.

Также точно: `Γ = N₃₀ + 4·N₃₃` (тождество); `N₀₃ ≈ N₃₀` (± симметрия, ~0.2%).

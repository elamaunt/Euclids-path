# 03. Активное просеивание до корня

> Lean-аналог: `EuclidsPath/Step03_ActiveSieve.lean` (заготовка)
> Источник: BE_UPDATED §3–4 (Теорема 3.1, Лемма 4.2).

## Цель шага

Связать «выживание» центра при просеивании до корня с простотой обоих чисел `6m±1`. Это критерий, по
которому выживший центр сразу даёт пару близнецов.

## Определения

- **Активный удар** clock `q` по индексу `k` — попадание `k` в запрещённый класс `q`.
- Множества активных прогрессий: `𝓗_q^+ = { (q²−1)/6 + tq : t ≥ 0 }`, аналогично `𝓗_q^−`.
- **Survivor до уровня `z`**: `𝓢_z = { m : ни один clock q ≤ z не бьёт m }`.

## Формулировки

**Теорема 3.1 (Active Twin Hit).** Пара `(6k−1, 6k+1)` не является парой близнецов тогда и только
тогда, когда `k` попадает хотя бы в одну активную прогрессию `𝓗_q^±` с `q ≤ √(6k+1)`.

**Лемма 4.2 (просеивание до корня даёт близнецов).** Если `m ∈ 𝓢_z` при `z = √(6m+1)` (точнее
`z ≥ √(6m+1)`), то оба числа `6m−1`, `6m+1` простые.

## Идея доказательства

Если оба `6m∓1` не имеют простых делителей `≤ √(6m+1)`, то каждое из них либо простое, либо `1`;
для `m` в рабочем блоке — простое. Обратно, составное число имеет простой делитель не больше корня,
который и есть бьющий clock.

## Статус

🟢 **строго.** Классический ситовой критерий (определение простоты через делители до корня).

# RESULTS: old-peel замыкание SNOL (sign law, height drop, regeneration)

Харнесс: `oldpeel_harness.py`. На 3000 реальных rank-1 catch (A=200): `6n+ε=a` простое `>A`,
`6n−ε` составное, пойманное малым `p≤A`; раскрытие `6n−ε = p(6t+δ)`.

## Числа — все три закона на 100%

| Закон | результат |
|---|---|
| sign law `δ = −π·ε` (`p≡π mod 6`) | **3000/3000 = 1.0000** |
| height drop `t < n/5` | **3000/3000 = 1.0000** |
| regeneration `t > 0` (поток продолжается) | **3000/3000 = 1.0000** |

Примеры (n,ε,a,p,δ,t,n/5): (6668,1,40009,11,1,606,1333), (6669,−1,40013,5,−1,1334,1334), …

## Смысл

Catch `p∣a−2ε` — **НЕ терминал**, а old-peel шаг спуска, строго уменьшающий высоту (`t<n/5`).
Значит «нет терминала ⟹ бесконечный спуск», что запрещено двигателем Евклида (EPMI). Это замыкает
финальный узел SNOL на **уже доказанную** невозможность двигателя — без счёта/распределения.

## Где вход (честно)

Алгебра old-peel (sign law, height drop) и замыкание `беск. спуск ⟹ False` — **доказаны в Lean**
(`Engine/OldPeel.lean`, `no_infinite_old_peel`, `old_peel_terminates`). Открытым входом остаётся
**NOPSL** (§11): что quotient `t` ВСЕГДА попадает в одну из категорий (clean-возврат / следующий
old-peel / fan-in-Hall / known-defect) — т.е. rigid-ledger замкнут относительно old-peel quotients
без «несclassifiable терминала». Это структурный (не счётный) вход — центральная точка аудита.

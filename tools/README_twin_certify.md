# twin_certify — детерминированный сертификатор близнецов из доказанных законов

Инструмент-следствие машинно-проверенных лемм программы. **Не** ускоряет факторизацию и **не**
угрожает RSA — он сертифицирует пары близнецов в своей нише быстрее и строже, чем вероятностные тесты.

## На чём основан (что доказано в Lean)

`EuclidsPath/Engine/Residuals.lean`, теорема **`oldfree_below_sq_prime`** (стандартные аксиомы, без `sorry`):

> Если ни один простой `p`, `5 ≤ p ≤ A`, не делит `6m−1` и `6m+1`, и `6m+1 < A²`,
> то `6m−1` и `6m+1` **оба простые**.

Плюс channel-law колесо (`PaymentLedger.lean`): центр-близнец избегает `m ≡ ±6⁻¹ (mod q)`.

Это даёт **детерминированный сертификат** простоты пары через **одно сито** до `A=⌈√(6m+1)⌉`,
амортизированное по всему окну, без отдельного теста простоты/факторизации каждой стороны.

## Вход / использование

Вход — **центр `m`** (пара = `(6m−1, 6m+1)`), один или окном.

```
python twin_certify.py certify 18            # один центр -> OK (107,109) / NO делитель
python twin_certify.py window 1000000 1001000  # все близнецы-центры в окне
python twin_certify.py list 1 18 4 2         # список центров
python twin_certify.py bench 3000000         # бенчмарк против baseline
```

## Ниша (где это лучше альтернатив) — с замерами

| режим | лучший метод | почему |
|---|---|---|
| **разреженные кандидаты в большом окне** | **этот инструмент** | одно сито до `√(6·hi)` амортизируется; **детерминированно** |
| нужен **сертификат**, не вероятность | **этот инструмент** | даёт доказательство простоты; Miller-Rabin — лишь вероятность |
| все близнецы подряд в окне | полное сито Эратосфена | сито диапазона быстрее для плотного перебора |
| одиночное гигантское число (RSA-масштаб) | — | упирается в `√n`, как trial-division (не ниша) |

Замер (`bench 3000000`, числа до 1.8·10⁷, 20000 разреженных центров):
- **лемма-сертификат: ~4.3× быстрее Miller-Rabin x2**, результат **идентичен**, и это **доказательство**, а не вероятность;
- полное сито выигрывает только в режиме «все подряд».

## Граница применимости (честно)

Лемме нужны простые до `A=√(6m+1)`. Практический предел — `A ≲ 2·10⁸`, т.е. центры `m` примерно
до **10¹⁵** (числа до ~10¹⁵). Для больших — инструмент **отказывает** (`size_guard`), потому что
он не обходит барьер `√n`. Это не алгоритм факторизации и не ускорение для гигантских чисел.

## Что это даёт исследователю

- **Воспроизводимый** детерминированный сертификат пар близнецов, обоснованный машинно-проверенной
  леммой (а не эвристикой);
- в нише разреженной проверки — **быстрее вероятностного Miller-Rabin** и при этом строже (сертификат);
- честные границы: ниша и предел `√n` явные, без преувеличений.
